# Regras Gerais do Projeto

## 1. Ponto de Partida
- O desenvolvimento **SEMPRE** começa pelo arquivo `AGENTS.md` na raiz do projeto. Ele é o orquestrador principal e contém o contexto global.

## 2. Estrutura de Documentação
- A pasta `Docs/` centraliza toda a documentação do projeto.
- **`Docs/WORKFLOWS.md`**: Descreve a metodologia de desenvolvimento (PREVC) e os fluxos de trabalho padrão. Siga-o rigorosamente.
- **`Docs/PROGRESS.md`**: Rastreia o andamento das implementações e o roadmap do projeto.
- **`Docs/BUGS.md`**: Documenta todos os bugs encontrados, suas soluções e impactos. Consulte-o antes de iniciar correções.
- **`Docs/Workflows/`**: Contém fluxos de trabalho detalhados para tarefas específicas (ex: debugging, segurança).

## 3. Princípio da Neutralidade
- Todos os arquivos nesta estrutura são templates. Ao iniciar um novo projeto, preencha os placeholders (`[PLACEHOLDER]`) e adapte o conteúdo conforme necessário, mas não altere a estrutura base dos templates.
